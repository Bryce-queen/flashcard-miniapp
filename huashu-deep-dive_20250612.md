## 二、huashu-md-html（反 AI slop Markdown → HTML）

### 新发现：微信专用模板 `templates/wechat/`
这是**直接对标闪卡产品场景**的资源。

#### 设计系统
| 元素 | 值 |
|------|-----|
| 底色 | `#f8f3e7`（比我们的 #fbf7ee 更暖一点）|
| 正文字色 | `#1f1a14` |
| 强调色 | `#c25b30`（terracotta） |
| 正文字体 | `Songti SC` 衬线体 — **不是无衬线！** |
| UI 字体 | `PingFang SC` 无衬线 |
| 标题 | Songti SC Semibold, 2-3rem, letter-spacing -0.012em |
| eybrow | PingFang SC 600, letter-spacing 0.22em, 赤陶色 |
| 行高 | 1.78 |

#### 微信兼容机制（template.html5 的 JS）
1. Post-process DOM：把 `::before` 伪元素转为真实 `<span>` 元素
2. H2 编号用 `<span class="h2-num">` 而非 CSS counter
3. UL bullet 用 `<span class="bullet">—</span>` 而非 `::marker`
4. Copy 时遍历 computed style，内联 34 个 CSS 属性
5. 使用 `ClipboardItem` API 同时写入 text/html + text/plain

#### 花叔 WeChat 排版铁律
- **CJK 衬线优先** — Songti SC 做正文字，不是 PingFang！
- **序号化章节** — 01/02 前缀而非 emoji
- **暖底 + 强调线** — 30% 宽 terracotta hr
- **bold 高亮** — `background: linear-gradient(to bottom, transparent 62%, #f3deb6 62%)`
- **号码用 tabular-nums** — `font-feature-settings: "tnum"`

#### 其他模板
- `article` — Warm Editorial 新闻体
- `interactive` / `reading` / `report` — 更多变体

### 对闪卡的启示
- ⚠️ **Canvas 可用宋体吗**？微信小程序 Canvas 2D 不支持外部字体。PingFang SC 是安全选择。但若未来做公众号图文版，应切换为 Songti SC。
- ✅ 我们的 30% 宽规则线 + terracotta 强调完全对标
- ✅ 底部签注格式一致

## 三、huashu-skills（22 个技能，原名 huashu-skills 非 hiashu-skills）

### 直接相关
- **huashu-wechat-image** — 公众号配图工作流
- **huashu-xhs-image** — 小红书配图工作流

### 花叔生产级配图工作流（提取精华）

#### 核心原则：先提案，后生成
```
理解内容 → 设计提案（2-3 方向）→ 用户选择 → 生成 → 预览确认
```

#### 两条路径对比
| | Path A: HTML→截图 | Path B: AI 生成 |
|---|---|---|
| 文字准确度 | 100% | 中文可能出错 |
| 视觉质感 | ⭐⭐（花叔：太平面） | ⭐⭐⭐⭐⭐ |
| 成本 | 零 | API 费用 |
| **花叔偏好** | ❌ 仅精确数据表格兜底 | ✅ **默认走 AI** |

#### 花叔公开审美画像
**喜欢**：纸张质感、手写笔触、印章、胶带、文字撑满画面、暖色调
**不喜欢**：HTML 截图（像 PPT 模板）、赛博霓虹/#0D1117、署名/水印、过度留白

#### 小红书 3:4 标准（1080×1440）
- 标题占画面 30-50%
- 核心数字放大 3 倍做视觉 hero
- 文字撑满，宁可满不要空

#### 公众号封面 2.35:1（1800×766）
- 安全区 766×766 中央方块
- 封面标题 ≤ 10 字

#### 深色模式适配
- 底色用 `#F5F5F5` 代替纯白
- 底色用 `#1A1A2E` 代替纯黑
- 避免阴影、白边框
- 低饱和度配色
